1. What is a Database?

A database is an organized collection of structured information, or data, typically stored electronically in a computer system. It is managed by a Database Management System (DBMS).

The Key Distinction

Data vs. Information:
"100, Apple, Red" is just data.
"Inventory: 100 Red Apples" is information derived from a database.

2. Why are Databases Important?

Databases solve the issues of redundancy (duplicates) and inconsistency found in manual filing systems.

Data Organization

Data is structured into rigorous formats (Schema). This ensures a phone number field can only hold numbers, preventing data corruption and ensuring "Clean Data" for analytics.

Efficiency & Speed

Using techniques like Indexing, databases can find 1 specific record among 100 million records in milliseconds. Manual search would take years.

Security & ACID

ACID properties (Atomicity, Consistency, Isolation, Durability) ensure that financial transactions are safe. If the power fails during a bank transfer, the database ensures money isn't lost.

3. Visualizing The Relational Model

In a Relational Database, data is stored in tables that are linked via "Keys". This reduces data duplication.

CUSTOMERS
PK Customer_ID INT
Full_Name VARCHAR
Email VARCHAR
Unique identifier for every user
ORDERS
PK Order_ID INT
FK Customer_ID INT
Order_Date DATE
Total_Amount DECIMAL
Links back to Customers Table

This relationship is One-to-Many:
"One Customer can have Many Orders, but one Order belongs to only One Customer."
